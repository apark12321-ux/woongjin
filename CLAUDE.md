# CLAUDE.md — 웅진 수학 문항 개발 파이프라인

> 이 파일은 Cowork / Claude Code가 이 폴더를 열 때 **매번 자동으로 읽는** 프로젝트 헌법이다.
> 새 대화마다 규칙을 다시 설명할 필요 없다. 아래 규칙과 프로세스를 항상 따른다.

---

## 0. 이 프로젝트가 하는 일

웅진씽크빅 외주 **수학 시험지(문제지 + 정답해설지)** 를 원본 PDF로부터 **HWPX**로 조판·출력한다.
- 입력: 원본 PDF + 선별 PDF(문항선별 또는 정답선별)
- 출력: `output/*.hwpx`
- **이미지는 사용자가 수작업으로 직접 삽입한다. Claude는 자리(placeholder)만 확보한다.**

---

## 1. 절대 원칙 (위반 금지)

1. **원문 그대로.** 숫자·문자·식을 임의로 변경·추정·계산으로 채우지 않는다.
2. **판독 불가 → `〔★확인필요〕` 표시** 하고, `output/`에 `★확인필요_목록.md`로 따로 정리한다.
3. **텍스트 레이어를 신뢰하지 않는다.** 반드시 pymupdf로 원본 PDF를 렌더해 **눈으로 대조**한다.
4. **생성형 이미지 금지.** 도해가 필요하면 코드 기반 렌더(figbase)만 사용한다.
5. **PDF offset = 0.** 선별표 쪽번호 = PDF 인쇄 쪽번호.

## 1-1. 컨텍스트 오버플로우 방지 (중요)

원본 PDF **전체를 컨텍스트에 올리지 않는다.** 채팅 업로드가 아니라 로컬 파일이므로:
- 선별표에서 대상 문항의 쪽번호를 먼저 파악한다.
- 해당 **페이지만** pymupdf로 고해상도 렌더(SC 크게)해서 확인한다.
- 100페이지·100이미지짜리 PDF도 페이지 단위로 처리한다. 한 번에 다 열지 않는다.

## 2. 작업 프로세스 (SOP)

모드는 두 가지, 흐름은 동일하다:
- **문제지 모드** — 입력: 원본 PDF + **문항선별** PDF
- **정답해설 모드** — 입력: 원본 PDF + **정답선별** PDF

| 단계 | 할 일 | 산출/체크 |
|---|---|---|
| ① 입력 확인 | `input/원본/`·`input/선별/` PDF 확인. 선별표에서 대상 문항·쪽번호 파악(offset 0) | 대상 목록 확정 |
| ② 원문 정독 | 선별된 **페이지만** pymupdf 렌더 → 문항·보기·도형·수식 원문 확인 | 판독불가 → `★확인필요` |
| ③ HWPX 조판 | `scripts/` 파이프라인으로 XML 수술: 수식 조판 규칙 적용, 보기 삽입, 도해는 figbase | 라운드트립 규칙 준수 |
| ④ 출력·검증 | `output/`에 HWPX 저장. 열림(라운드트립) 검증 + 수식 전수검사 + ★목록 정리 | 이미지 자리만 확보 |

## 3. 수식 조판 규칙 (핵심 요약)

> 208개 샘플 실측으로 확정된 규칙. 상세·예시는 `rules/수식조판_규칙.md` 참조.
> 한글을 제외한 전부는 수식편집기(Ctrl+N,M / 10pt). **수식 안에 한글 금지.**

| 항목 | 규칙 | 예 |
|---|---|---|
| = 분할 | 관계연산자 `=`에서 분할하고 `=`는 **우변 개체에 붙임**(HEAD 모드). `=`마다 분할 | `AB │ =AC`, `∠B │ =70°`, `∠x │ =90°−36°−36° │ =18°` |
| 산술·비 | 산술(±·×·÷·분수)·비(`:`)는 한 개체에 **통째** 유지 | `x+y` 통째, `∠x:∠y:∠z │ =3:7:5` |
| ⊥·≡ 병합 | `PERP`·`EQUIV`는 **좌변 피연산자와 한 개체로 병합**(조각나면 기호로 안 뜸) | `l PERP n`, `bar{rmAB} PERP bar{rmDE}`, `TRIANGLE rmABC EQUIV TRIANGLE rmDCB` |
| 대문자 표기 | `ANGLE / TRIANGLE / PERP / EQUIV / TIMES / DEG` 은 대문자 | |
| 기호 | `bar{}`=선분, `vec{}`=반직선, `dyad{}`=직선, `{a}over{b}`=분수, `rm`=점이름·단위·합동·면 | |
| 이탤릭 | 소문자 변수·직선이름(`l·m·n`)은 이탤릭(rm 안 붙임) | |
| 호(arc) | `rm arch XX`(정체). 명사 '호'는 텍스트로 유지 | |
| 숫자-단위 | 숫자와 단위 사이 **1칸** 띄움 | `9 rm cm` (`9rmcm` 아님) |
| 간격 | 콤마 뒤 `~`(한 칸), 반 칸은 `` ` ``(백틱) | |

## 4. HWPX 라운드트립 규칙 (핵심)

> 상세는 `rules/HWPX_라운드트립.md` 참조.

- `mimetype`은 **STORED(무압축), 첫 번째 엔트리**로 재패키징.
- 원본 zip의 **엔트리 순서를 유지**한다.
- XML은 **strict 파싱**.
- lxml 재직렬화 시 **XML 선언 따옴표·ZipInfo 메타데이터 불일치**가 나면
  한글에서 "문서가 손상되었거나 변조되었을 가능성" 오류 → 원본 메타데이터를 보존해 재패키징.
- 스택: `lxml 6.0.2` + Python `zipfile` + `pymupdf`.

## 5. 폴더 규칙

```
웅진/
├─ CLAUDE.md              ← 이 파일 (자동 로드)
├─ rules/                 ← 상세 규칙 문서
├─ scripts/               ← 파이썬 파이프라인 (아래 참조)
├─ input/원본/            ← 이번 작업 원본 PDF 드롭
├─ input/선별/            ← 이번 작업 선별 PDF(문항/정답) 드롭
├─ output/               ← HWPX 출력물 + ★확인필요_목록.md
├─ work/                 ← 렌더 이미지 등 중간 산출물
└─ 정답해설/              ← 정답해설 PDF 보관
```
