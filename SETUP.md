# SETUP — 로컬 연속 작업 환경 세팅

이 폴더를 로컬(예: `D:\Users\jmvh\문서\웅진\`)에 두고 **Cowork 또는 Claude Code**로 여는 순간,
`CLAUDE.md`가 자동 로드되어 규칙·프로세스를 다시 설명할 필요가 없어진다.
PDF는 채팅에 업로드하지 않고 **폴더에 드롭**하므로 컨텍스트 오버플로우가 없다.

---

## 1단계 — 폴더 배치

이 스타터의 압축을 풀어 원하는 위치에 둔다. 폴더 구조:

```
웅진/
├─ CLAUDE.md
├─ SETUP.md
├─ rules/  scripts/  input/원본/  input/선별/  output/  work/  정답해설/
```

## 2단계 — 기존 스크립트 넣기

`scripts/`에 기존 파이썬 파이프라인을 복사한다:
`hwpx_eqn.py`, `figbase.py`, `build_base_v3.py`, `woongjin_layout_fix.py`
(없으면 첫 작업 때 규칙에 맞춰 새로 만들면 된다.)

## 3단계 — 앱으로 폴더 열기

- **Cowork**(권장, 비개발 워크플로): 데스크톱 앱에서 이 폴더를 작업 폴더로 지정.
- **Claude Code**: 이 폴더에서 `claude` 실행.

둘 다 로컬 파일을 직접 읽고, `scripts/`의 파이썬을 실행하고, `output/`에 HWPX를 쓴다.

## 4단계 — (선택) GitHub Desktop 연동

이 폴더를 GitHub Desktop 리포로 만들면 **규칙·스크립트가 버전 관리**된다.
`input/`·`output/`·`work/`는 `.gitignore`로 빼는 걸 권장(대용량 PDF/HWPX).

---

## 새 작업 시작하는 법 (매번 이것만)

1. 이번 시험지 **원본 PDF → `input/원본/`**, **선별 PDF → `input/선별/`** 에 드롭.
2. 앱에서 한 줄:

   > `input`에 이번 작업 PDF 넣었어. CLAUDE.md 규칙대로 HWPX 만들어서 `output`에 저장하고,
   > 판독 안 되는 건 ★확인필요 목록으로 정리해줘. 이미지는 내가 넣을 테니 자리만 잡아줘.

3. `output/`에서 결과 HWPX와 `★확인필요_목록.md` 확인 → 한글에서 열어 검증 → 이미지 수작업 삽입.

끝. 규칙 설명도, PDF 채팅 업로드도 없다.

---

## 왜 이 방식이 오버플로우를 막나

채팅창은 업로드한 PDF의 모든 페이지 이미지를 컨텍스트에 올리려다 100장짜리에서 터진다.
로컬 작업에서는 pymupdf로 **대상 문항 페이지만** 렌더해 확인하므로,
페이지 수·이미지 수와 무관하게 컨텍스트가 안정적이다.
