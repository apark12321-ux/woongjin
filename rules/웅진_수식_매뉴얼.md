# 웅진 빈출유형 — 수식 매뉴얼

> 여러 대화에 흩어져 있던 수식 규칙을 **웅진 빈출유형 기준으로 통일**한 문서입니다.
> (쎈 엔진 전용 규칙은 파이프라인이 달라 이 문서에서 제외했습니다.)
> **숫자–단위 간격 = 1칸 확정**(§9).

---

## 1. 절대 원칙
- **원문 그대로.** 숫자·문자·식을 임의로 변경/추정/계산으로 채우지 않는다.
- 판독 불가 = **〔★확인필요〕** 표시 후 별도 체크 목록.
- **한글을 제외한 모든 영문·숫자·기호·괄호·단위 = 무조건 수식**(Ctrl+N,M / 10pt).
- 수식 편집기 **안에 한글 절대 금지**. 한글을 감싸는 괄호는 괄호만 수식(`(참)` → 수식`(` + 글자`참` + 수식`)`).
- 수학 기호·괄호·단위는 **수식 편집창 명령만** 사용(문자표 수학기호 금지). 원문자 ①~⑮만 문자표.

---

## 2. 한컴 수식 스크립트 기본 문법

| 기능 | 명령 | 예 |
|---|---|---|
| 분수 | `{A} over {B}` | `{1} over {2}` |
| 위첨자 / 아래첨자 | `A ^{n}` / `A _{n}` | `x ^{2}`, `a _{1}` |
| 제곱근 | `sqrt {A}` | `sqrt {3}` |
| 켤레·오버라인 | `BAR{z}` / `OVERLINE{식}` | `BAR{z}` |
| 자동크기 괄호 | `left ( 식 right )` | |
| 빈칸 | `~`(한 칸) · `` ` ``(반 칸) | |
| 줄바꿈 | `#` | |
| ± · × · ∴ · ≠ · … | `pm` · `TIMES`/`cdot` · `therefore` · `ne` · `cdots` | |

**글자장식(accent) 전체 목록:** `HAT · CHECK · TILDE · ACUTE · GRAVE · DOT · DDOT · BAR · VEC · DYAD · UNDER`
→ **호(⌒)는 accent 명령이 없다**(§5에서 `rm arch`로 처리). `arc`는 arcsin·arccos 계열 함수명일 뿐.

---

## 3. 수식화 범위
- 점 라벨(A·B·C·O·P…), 변수(a·b·c·x·y·k·m·n…), 도형명(△·□·◇), 단위, 괄호, 기호 → **전부 «수식»**.
- 콤마로 구분된 점이름 목록은 **각각 개별 수식 객체**: "A, B, C, D" → `rmA` / `rmB` / `rmC` / `rmD` (콤마·공백은 텍스트).

---

## 4. 직립(rm) / 이탤릭
- **숫자** = 그대로(직립, rm 안 붙임).
- **대문자 점이름·단위·합동/면 이름 = `rm`(정체)**: `rmA`, `rmAOC`, `rmcm`.
- **소문자 변수(x, y, a, b) = 이탤릭**(rm 안 붙임).
- **line 이름 l·m·n = 이탤릭으로 통일**(단, 면·단위 rm은 보존).

---

## 5. 선분·직선·반직선·호

| 구분 | 명령 | 예 |
|---|---|---|
| 선분 (—) | `bar{ }` | `bar{rmAB}` |
| 직선 (↔) | `dyad{ }` | `dyad{rmAB}` |
| 반직선 (→) | `vec{ }` | `vec{rmAB}` |
| **호 (⌒)** | **`rm arch XX`**(정체) | `rm arch AB` · `rm arch BC` · `rm arch CA` |

> 호는 **원과 부채꼴** 단원에만 적용. 명사 '호'("호의 길이", "호로 이루어진")는 **텍스트로 유지**.

---

## 6. 관계·기호 명령 (대문자 표기)
`ANGLE · TRIANGLE · PERP(⊥) · EQUIV(≡) · TIMES · DEG` — **전부 대문자.**
- 평행 = 수식 `//` (⫽)
- 삼각비 = `sin rmA` / `cos rmA` / `tan rmA`
- 도 = `^{circ}` 또는 `DEG`

---

## 7. `=` 기준 분할 (HEAD 모드) ★핵심
**관계연산자(=) 앞에서 분할하고, `=`는 우변 개체에 붙인다. 산술은 통째.**

| 입력 | 분할 결과 | 설명 |
|---|---|---|
| `AB=AC` | `AB` │ `=AC` | `=` 앞 분할, `=`는 우변에 부착 |
| `∠B=70°` | `∠B` │ `=70°` | 좌변 / =우변 |
| `∠x=90°−36°−36°=18°` | `∠x` │ `=90°−36°−36°` │ `=18°` | `=`마다 분할, 산술 `−`은 개체 안 유지 |
| `x+y` | `x+y` | 산술(+ − × ÷ 분수)은 분할 안 함 |
| `∠x:∠y:∠z=3:7:5` | `∠x:∠y:∠z` │ `=3:7:5` | 비(:)는 유지, `=`만 분할 |
| `∠AOC=∠BOD=90°` | `∠AOC` │ `=∠BOD` │ `=90°` | 3조각 |

- 분할점은 관계연산자 앞뿐: `=` `≡` `⊥` `∥` `<` `>` `<=` `>=` `!=` `sim` (n개 → 조각 n+1개, **길이 예외 없음**).
- **`+ − × ÷ · :` 에서는 절대 분할 금지.** 항·비는 한 덩어리.
- 중괄호 `{ }` 안(분수·첨자 내부) 연산자는 분할 대상 아님.
- 조각 사이 **공백 없이** 붙임 → 화면상 붙어 보임.

---

## 8. `⊥`(PERP)·`≡`(EQUIV) 좌변 병합
관계연산자가 왼쪽 피연산자와 **떨어져 조각나면 한 개체로 병합**(조각나면 기호로 안 뜸).

- `⟨l⟩⟨PERP n⟩` → `l PERP n`  (→ `l ⊥ n`)
- `⟨bar{rmAB}⟩⟨PERP bar{rmDE}⟩` → `bar{rmAB} PERP bar{rmDE}`  (→ `AB̄ ⊥ DĒ`)
- `⟨TRIANGLE rmABC⟩⟨EQUIV …⟩` → `TRIANGLE rmABC EQUIV TRIANGLE rmDCB`  (→ `△ABC ≡ △DCB`)

---

## 9. 띄어쓰기 — **숫자–단위 1칸 확정**
- **숫자와 단위 사이 = 1칸 띄움**: `9 rm cm`, `20 rm cm`, `180 DEG`.
  - ❌ 붙여쓰기(`9rmcm`) 금지.
- **콤마(,) 뒤 = 한 칸 `~`**: `1, ~ 2, ~ 3`.
- **반 칸 = `` ` ``(백틱)**: 위/아래 첨자 주변, 변수와 예약어 사이.

---

## 10. HWPX 라운드트립 안전수칙 (수식 개체 편집 시)
"문서가 손상되었거나 변조" 오류를 막는 규칙.

- **mimetype = STORED(무압축)로 첫 엔트리.**
- **엔트리 순서 유지**(원본 zip namelist 순서 그대로).
- **strict XML 파싱** 통과 확인.
- 수식/표 개체 복제 시 **ID·zOrder 값 전부 새로 부여**(충돌 방지).
- 수식 개체 복제는 **같은 파일 안에서만**(charPr/borderFill ID가 파일마다 달라 교차 복제 금지).
- 검증: XML strict ✓ / ID·zOrder 중복 없음 ✓ / paraPr·charPr·borderFill 참조 정의됨 ✓ / mimetype STORED 첫 엔트리·재파싱 ✓ / 이미지 MD5 보존 ✓ / ★잔여 0.
