# HWPX 라운드트립 규칙 (전문)

HWPX는 ZIP + XML 구조다. XML을 수술한 뒤 **원본과 동일한 방식으로 재패키징**하지 않으면
한글에서 "문서가 손상되었거나 변조되었을 가능성이 있습니다" 오류가 난다.

## 스택
- `lxml 6.0.2` — XML 파싱/재직렬화
- Python `zipfile` — ZIP 재패키징
- `pymupdf` — 원본 PDF 렌더(눈 대조용)

## 필수 규칙

1. **mimetype 엔트리**
   - `mimetype`은 **첫 번째 엔트리**여야 하고, **STORED(무압축)** 로 저장한다.

2. **엔트리 순서 보존**
   - 원본 zip의 엔트리 순서를 그대로 유지해 다시 쓴다. 순서가 바뀌면 오류.

3. **strict XML 파싱**
   - 관대(recover) 파싱 금지. strict로 파싱해 구조 손상을 조기 발견한다.

4. **XML 선언 따옴표 일치**
   - lxml 재직렬화 시 XML 선언의 따옴표 스타일이 원본과 달라지면 변조로 인식될 수 있다.
   - 원본 선언을 보존하거나 동일 스타일로 맞춘다.

5. **ZipInfo 메타데이터 일치**
   - 각 엔트리의 압축 방식·메타데이터를 원본과 맞춘다. 불일치 시 손상 오류.

## 안전 재패키징 패턴 (개념)

```
원본 zip 열기
→ 각 엔트리를 원본 순서대로 순회
→ 대상 XML만 lxml로 파싱·수정·재직렬화(선언/인코딩 보존)
→ 나머지 엔트리는 바이트 그대로 복사
→ mimetype은 STORED로 첫 번째에 기록
→ 나머지는 원본과 동일한 압축 방식으로 기록
```

## 검증
- 저장 후 한글에서 **정상 열림** 확인(라운드트립 검증).
- 이미지 삽입 전후 MD5 대조로 이미지 무결성 확인.
